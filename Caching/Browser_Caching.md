# 🚀 Browser Caching

## What is Caching?

Caching is the process of storing frequently used resources so they can be reused without downloading them again.

### Benefits

* Faster page loads
* Reduced network requests
* Reduced server load

---

# Browser Cache

When the browser downloads a resource (JS, CSS, image, etc.), it stores:

* Resource (file)
* Response headers
* Metadata (ETag, Cache-Control, Last-Modified, etc.)

On future requests, the browser automatically decides whether to use the cached resource or contact the server.

---

# Cache-Control

`Cache-Control` tells the browser how to cache a resource.

### max-age

```http
Cache-Control: max-age=60
```

* Resource can be used directly from cache for **60 seconds**.
* No request is sent to the server during this period.

### no-cache

```http
Cache-Control: no-cache
```

* Resource **can be cached**.
* Browser must validate with the server before using it.

### no-store

```http
Cache-Control: no-store
```

* Resource is **never cached**.
* Browser downloads it on every request.

---

# ETag

ETag is a **unique identifier (fingerprint)** generated by the server for a resource.

Example:

```http
ETag: "abc123"
```

After cache expires, the browser sends:

```http
If-None-Match: "abc123"
```

The server compares the ETag:

* Same → `304 Not Modified` → Browser uses cached file.
* Different → `200 OK` → Browser downloads the new file.

---

# Last-Modified

Instead of comparing a fingerprint, the server compares the **last modified timestamp**.

Server sends:

```http
Last-Modified: Wed, 02 Jul 2026 10:30:00 GMT
```

Browser later sends:

```http
If-Modified-Since: Wed, 02 Jul 2026 10:30:00 GMT
```

Server compares the timestamp:

* Same → `304 Not Modified`
* Changed → `200 OK` + new resource

---

# ETag vs Last-Modified

| ETag                     | Last-Modified            |
| ------------------------ | ------------------------ |
| Uses fingerprint/version | Uses timestamp           |
| More accurate            | Less accurate            |
| Uses `If-None-Match`     | Uses `If-Modified-Since` |

---

# Important Interview Points

* Cache expiration **does not** mean the browser immediately downloads the file again.
* The browser first validates the cached resource using **ETag** or **Last-Modified**.
* The browser automatically manages HTTP caching; developers do not write code for this behavior.
* ETag and Last-Modified are stored as part of the cached resource's **metadata** in the browser's HTTP cache.
* In React production builds, hashed filenames (e.g., `main.82fd3.js`) act as **cache busting**, so the browser requests a new file instead of validating the old one.

---

# Quick Revision

* **Browser Cache** → Stores resources and metadata.
* **Cache-Control** → Defines caching rules.
* **max-age** → Use cache until it expires.
* **no-cache** → Cache it, but validate before using.
* **no-store** → Never cache.
* **ETag** → Fingerprint of a resource.
* **If-None-Match** → Browser asks if the ETag is still valid.
* **Last-Modified** → Last update time of a resource.
* **If-Modified-Since** → Browser asks if the resource changed since the stored timestamp.
* **304 Not Modified** → Use cached resource.
* **200 OK** → Download updated resource.
